### Фильтрация и сглаживание
1. Сглаживание выполнялось через свёртку: cv2.filter2D(dx_array, -1, kernel) и np.convolve для tx/ty. Цель: уменьшить резкие скачки смещения и сгладить траекторию камеры.
2. Warping видео:
  - Farnebäck: cv2.remap использует плотный поток для смещения каждого пикселя.
  - LK: cv2.warpAffine применяет аффинное смещение, вычисленное по разреженным точкам.

### Основные наблюдения
1. Стабилизация по метрикам
 - Метод Farnebäck уменьшил межкадровые изменения.
 - Метод Lucas–Kanade в данном эксперименте показал хуже оригинала, добавляя ошибки в движение.
2. Причины различий
 - Быстрое движение камеры, затрудняющее корректное определение движения пикселей.
 - Однородные поверхности (стены, небо, дорога), из-за этого мало характерных точек, LK хуже оценивает движение.
 - Движущиеся объекты (машины), алгоритм может принять их движение за движение камеры.
 - Farnebäck устойчив, т.к. учитывает движение всех пикселей, а LK сильно зависит от качества и числа feature points.
